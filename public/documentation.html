Bussiness Logic URL HTTP Method Request body data format Response body data format
Get a list of Movies /movies GET None A JSON object holding data about all the movies
Get data about a single Movie by title /movies/[title] GET None A JSON object holding data about a single movie, containing an ID, title, description, genre, imageURL, actors, and director. Example: 
{
    "id": "1",
    "title": "Scott Pilgrim vs. the World",
    "directed": "Edgar Wright",
    "description": "Scott Pilgrim must defeat his new...",
    "genre": [
        "Action",
        "Comedy",
        "Romance"
    ],
    "imageURL": "https://image.tmdb.org/t/....",
    "actors": [
        "Michael Cera",
        "Mary Elizabeth Winstead",
        "Kieran Culkin"
    ],
    "director": "Edgar Wright"
}
Get a list of movies by their genre /genres/[genre] GET None A JSON object holding data about movies with the genre passed
Get a list of Directors /directors GET None A JSON object holding data about all the directors
Get data about a single director by name /directors/[name] GET None A JSON object holding data about a single director, containing an ID, name, bio, birthYear, and deathYear. Example: 
{
    "id": 4,
    "name": "Christopher Nolan",
    "bio": "British-American filmmaker renowned for...",
    "birthYear": 1970,
    "deathYear": null
}
Create a user /regist POST A JSON object holding data about the user to add, strutured like this: 
{
    "name": "John",
    "username": "john123",
    "email": "test@email.com",
    "password": "password",
    "dateOfBirth": "1990-01-01"
}
A text message indicating that the registration was sucessfully
Get a list of users /users GET None A JSON object holding data about all the users
Update certan informations about a user by their id /users/[id] PUT A JSON object holding data about the user to update. Can update the username, email, password and dateOfBirth. Example: 
{
    "password": "NewPassword"
}
A text message indicating that the update was sucessfully
